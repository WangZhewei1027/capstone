Linked List — Singly Linked List (Interactive)

Explore nodes, insertions, deletions, and traversal with animated feedback.

Quick Overview

This module focuses on a single concept: singly linked lists and how nodes connect through "next" pointers. Use the controls to add, insert, remove and step through nodes to see how references change.

Controls

Select a node by clicking it (or Tab → Enter). Use Insert After to place a node after the selected node.

All operations announce changes via the log area and are keyboard-accessible.

Concept Title: Singly Linked List (nodes with a value and a next pointer)
Learning Objective: After using this module you will understand node relationships, how insertion and removal affect pointers, and how traversal visits nodes sequentially.

Interaction Design

- Click a node or use keyboard Tab + Enter to select it. Selected nodes are visually elevated.
- Add Head: creates a new node at the front; it animates into place.
- Append Tail: adds a node at the end; nodes shift and connecting arrows update.
- Insert After: insert a new node immediately after the currently selected node. If none is selected, the button is disabled.
- Delete Selected: removes the selected node with a fade/slide animation; pointers update accordingly.
- Traversal: Step advances a "current" highlight to the next node and announces the value. Play automatically steps through nodes with a chosen speed; Reset stops traversal and clears highlights.
- Visual feedback includes node insert/delete animations, a pulsing highlight for the current traversal node, and SVG arrows that update positions smoothly.

Layout Description

- Safe area: the entire layout uses a 24px padding on all sides. Controls and visuals use a minimum 16px spacing.
- The page uses a two-column layout: the left column contains controls and forms (fixed min-width 360px); the right column contains the interactive visual canvas and log.
- The canvas shows nodes in a horizontal row with arrows drawn via an SVG overlay. It is scrollable on small screens. Controls wrap under the visualizer for narrow viewports to maintain clarity and focus.
- Accessibility: controls have labels, nodes are keyboard-focusable, and user actions produce text updates in an aria-live log for screen readers.
