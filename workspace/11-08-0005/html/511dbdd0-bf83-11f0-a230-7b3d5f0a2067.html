# Interactive BFS Learning Module Design ## Concept Title **"Exploring Breadth-First Search: Layer by Layer Graph Traversal"** ## Learning Objective Users will understand how BFS systematically explores a graph by visiting all nodes at the current depth level before moving to nodes at the next depth level, visualizing the queue-based mechanism and recognizing BFS's ability to find shortest paths in unweighted graphs. ## Interaction Design ### User Interactions: 1. **Graph Building Mode**: - Click on empty canvas area to create new nodes - Click and drag from one node to another to create edges - Right-click on nodes/edges to delete them - Double-click on a node to set it as the start node (highlighted in green) 2. **Algorithm Controls**: - "Step Forward" button: Advances BFS by one step - "Auto Run" toggle: Automatically steps through at adjustable speed - "Reset" button: Clears the traversal state while keeping the graph - "Clear Graph" button: Removes all nodes and edges - Speed slider: Controls auto-run animation speed (100ms - 2000ms) 3. **Visual Feedback**: - **Node States**: - Unvisited (light blue) - In queue (yellow with pulsing animation) - Currently visiting (orange with expanding ring effect) - Visited (dark blue) - Start node (green border) - **Edge Animations**: - When traversing an edge, it highlights and shows a flowing particle effect - Discovered edges remain highlighted in blue - **Queue Visualization**: - Live queue display showing node order with smooth add/remove animations - Current node pops from front with slide animation - New nodes push to back with fade-in effect 4. **Information Panel**: - Real-time traversal order display - Current depth level indicator - Statistics: nodes visited, edges traversed, current queue size - Shortest path display when clicking any visited node ## Layout Description ### Spatial Organization: ``` ┌─────────────────────────────────────────────────────────────┐ │ BFS Interactive Explorer │ 24px margin ├─────────────────────────────────────────────────────────────┤ │ │ │ ┌─────────────────┐ ┌──────────────────────────────────┐ │ │ │ │ │ │ │ │ │ Control Panel │ │ Graph Canvas │ │ │ │ │ │ │ │ │ │ [Step Forward] │ │ (Interactive Graph Area) │ │ │ │ [Auto Run ◯] │ │ │ │ │ │ Speed: ▬▬▬○ │ │ │ │ │ │ [Reset] │ │ │ │ │ │ [Clear Graph] │ │ │ │ │ │ │ │ │ │ │ │ Instructions: │ └──────────────────────────────────┘ │ │ │ • Click to add │ │ │ │ • Drag to link │ ┌──────────────────────────────────┐ │ │ │ • Double-click │ │ Queue Display │ │ │ │ to set start │ │ Front → [A] [C] [D] ← Back │ │ │ │ │ └──────────────────────────────────┘ │ │ └─────────────────┘ │ │ │ │ ┌─────────────────────────────────────────────────────────┐ │ │ │ Traversal Info: │ │ │ │ Order: START → A → B → C → D │ │ │ │ Depth: 2 | Visited: 4/7 | Queue Size: 3 │ │ │ └─────────────────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────────────┘ ``` ### Key Layout Features: 1. **Left Panel (300px wide)**: - Fixed position with controls - 16px padding internally - Minimum 16px spacing between all buttons - Clear visual hierarchy with grouped controls 2. **Center Canvas (flexible width)**: - Takes remaining space minus panels - Minimum 500px width - Light grid background for spatial reference - Smooth pan/zoom capabilities 3. **Queue Display (below canvas)**: - Horizontal layout showing queue state - Fixed height of 60px - Visual arrows indicating front/back - Smooth transitions for enqueue/dequeue 4. **Info Panel (bottom)**: - Full width minus margins - Shows traversal statistics - Updates in real-time - High contrast for readability ### Responsive Design: - On screens < 768px: Stack control panel above canvas - Maintain 24px safe margins on all viewport edges - Scale node sizes based on viewport dimensions - Ensure touch-friendly interaction targets (minimum 44px) ### Visual Consistency: - Consistent color palette for algorithm states - Smooth transitions (0.3s ease-in-out) - Clear focus indicators for accessibility - High contrast text (WCAG AA compliant)
