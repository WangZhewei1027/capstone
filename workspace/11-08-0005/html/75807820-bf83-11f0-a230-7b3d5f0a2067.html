# Interactive Design Plan: Floyd-Warshall Algorithm ## Concept Title **"Finding All Shortest Paths: The Floyd-Warshall Algorithm"** ## Learning Objective Users will understand how the Floyd-Warshall algorithm systematically finds the shortest paths between all pairs of vertices in a weighted graph by considering intermediate vertices one at a time. They will visualize the dynamic programming approach and see how the distance matrix evolves through each iteration. ## Interaction Design ### User Interactions: 1. **Graph Creation Mode:** - Click on empty canvas to add vertices (nodes) - Click and drag between vertices to create edges - Click on edges to set weights via input field - Right-click vertices or edges to delete them 2. **Algorithm Execution:** - "Run Algorithm" button starts the visualization - "Step Forward/Backward" buttons for manual control - Speed slider to control animation pace - "Reset" button to return to initial state 3. **Visual Feedback:** - Current intermediate vertex (k) highlighted in distinctive color - Source vertex (i) and destination vertex (j) highlighted when checking paths - Edge animations showing path comparisons - Distance matrix cells animate when values update - Color coding: shorter path found (green), no improvement (red), current check (yellow) ### State Changes: - Distance matrix updates in real-time - Path reconstruction visualization - Step counter and algorithm state indicator - Highlighted explanation text for current operation ## Layout Description ### Spatial Organization: ``` +------------------[VIEWPORT]------------------+ | [24px margin] | | +------------ HEADER SECTION -------------+ | | | Title: Floyd-Warshall Algorithm | | | | Subtitle: Finding All Shortest Paths | | | +----------------------------------------+ | | | | +-- CONTROL PANEL --+ +-- GRAPH CANVAS --+| | | [Run Algorithm] | | || | | [Step Forward] | | Interactive || | | [Step Backward] | | Graph Area || | | [Reset] | | || | | Speed: [====o===] | | (450x400px) || | | Mode: ○ Edit | | || | | ○ View | | || | +------------------+ +-------------------+| | | | +------------ MATRIX DISPLAY -------------+ | | | Distance Matrix Path Matrix | | | | [Dynamic Grid] [Dynamic Grid] | | | +----------------------------------------+ | | | | +----------- EXPLANATION PANEL ----------+ | | | Current Step: k=2, i=1, j=3 | | | | Checking: dist[1][3] vs | | | | dist[1][2] + dist[2][3] | | | +----------------------------------------+ | | [24px margin] | +----------------------------------------------+ ``` ### Spacing Guidelines: - 24px safe margins on all viewport edges - 16px minimum spacing between all interactive elements - 32px spacing between major sections - Control buttons: 40px height with 8px padding - Matrix cells: 40x40px with 2px borders -
