# Interactive BFS Explorer - Design Plan ## Concept Title **"Breadth-First Search: Layer-by-Layer Graph Exploration"** ## Learning Objective Users will understand how BFS systematically explores a graph level by level, visiting all nodes at distance k before moving to nodes at distance k+1. They'll grasp the queue-based mechanism, see why BFS finds shortest paths in unweighted graphs, and recognize its breadth-first exploration pattern through interactive visualization. ## Interaction Design ### Core Interactions: 1. **Graph Building Mode** - Click on empty canvas area to create nodes - Click and drag from one node to another to create edges - Right-click nodes to delete them (and their connections) - Nodes automatically labeled with letters (A, B, C...) 2. **BFS Execution Mode** - Click "Start BFS" button to enter execution mode - Click any node to select it as the starting point - Use "Step" button to advance BFS one node at a time - Use "Auto-Run" to watch BFS execute automatically with adjustable speed - "Reset" button to clear the traversal state while keeping the graph 3. **Visual Feedback** - **Node States:** - Unvisited: Light gray with dark border - In Queue: Yellow with pulsing animation - Currently Processing: Bright blue with expanding ring effect - Visited: Dark blue with distance label - **Edge Highlighting:** - Edges light up when traversed - Tree edges shown in blue, non-tree edges in gray - **Queue Visualization:** - Live queue display showing current contents - Smooth animations for enqueue/dequeue operations 4. **Information Panel Updates** - Real-time display of: - Current node being processed - Distance from start - Parent node in BFS tree - Queue contents with visual representation ## Layout Description ### Spatial Organization: ``` ┌─────────────────────────────────────────────────────────────┐ │ Safe margin (24px) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ Title: "Breadth-First Search Explorer" │ │ │ │ Subtitle: "Build a graph and watch BFS in action" │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ ┌─────────────────┬───────────────────────────────────┐ │ │ │ │ │ │ │ │ Control Panel │ Graph Canvas │ │ │ │ │ │ │ │ │ [Start BFS] │ (Interactive graph area) │ │ │ │ [Step] │ │ │ │ │ [Auto-Run] │ │ │ │ │ Speed: [====] │ │ │ │ │ [Reset] │ │ │ │ │ [Clear Graph] │ │ │ │ │ │ │ │ │ │ Instructions: │ │ │ │ │ • Click to │ │ │ │ │ add nodes │ │ │ │ │ • Drag between │ │ │ │ │ nodes for │ │ │ │ │ edges │ │ │ │ │ │ │ │ │ └─────────────────┴───────────────────────────────────┘ │ │ │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ Queue Visualization: [A] → [C] → [D] │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ Current Status: │ │ │ │ Processing: Node B | Distance: 1 | Parent: A │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ └─────────────────────────────────────────────────────────────┘ ``` ### Key Layout Features: 1. **Left Control Panel** (200px wide) - Fixed position with all controls - Clear visual hierarchy with grouped buttons - Instructions always visible - 16px padding, 16px spacing between buttons 2. **Main Graph Canvas** (flexible width) - Takes remaining space - Light grid background for spatial reference - Smooth pan/zoom capabilities (optional enhancement) - Canvas boundaries clearly defined 3. **Bottom Information Panels** - Full width, fixed height (80px each) - Queue visualization with animated transitions - Status panel with key information - 16px spacing between panels 4. **Responsive Considerations** - Minimum viewport width: 768px - Graph canvas scales proportionally - Control panel remains fixed width - Font sizes scale appropriately ### Visual Hierarchy: - Primary focus: Graph canvas (largest area, centered) - Secondary: Active controls and queue visualization - Tertiary: Instructions and status information - Clear visual separation between sections using borders and background colors This design ensures users can easily build graphs, control the algorithm execution, and observe the breadth-first exploration pattern with clear visual feedback throughout the learning experience.
