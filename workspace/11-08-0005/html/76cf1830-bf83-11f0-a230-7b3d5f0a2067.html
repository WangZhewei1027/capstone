They'll learn to identify valid orderings, understand why cycles prevent topological sorting, and see how the algorithm works step-by-step through Kahn's algorithm. ### Interaction Design The interface presents a graph of tasks with dependencies that users can manipulate: 1. **Graph Manipulation:** - Click and drag nodes to reposition them for clarity - Click "Add Edge" button, then click two nodes to create a dependency - Click edges to remove them - Click "Add Node" to create new tasks 2. **Algorithm Visualization:** - "Run Topological Sort" button executes Kahn's algorithm step-by-step - Nodes highlight as they're processed (green when added to result, gray when completed) - In-degree counts display on each node and update in real-time - Queue visualization shows which nodes are ready for processing 3. **Visual Feedback:** - Hover effects on nodes and edges - Smooth animations when nodes are processed - Red highlighting when cycles are detected - Success/error messages with explanations 4. **Learning Features:** - Preset examples (course prerequisites, build dependencies) - "Create Cycle" button to demonstrate why cycles break topological sort - Speed control for algorithm animation - Step-by-step explanation panel ### Layout Description The page uses a three-panel layout: **Left Panel (300px):** - Controls section with buttons for graph manipulation - Directed edges shown as arrows - Queue visualization at the bottom - Minimum 16px spacing between all nodes **Right Panel (300px):** - Explanation section showing current algorithm state - In-degree table for all nodes -
