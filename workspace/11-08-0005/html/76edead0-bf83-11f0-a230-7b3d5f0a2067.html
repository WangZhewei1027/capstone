## Interactive Design Plan: Prim's Algorithm ### Concept Title **Prim's Algorithm: Building Minimum Spanning Trees Step by Step** ### Learning Objective Users will understand how Prim's algorithm constructs a minimum spanning tree by: - Starting from an arbitrary vertex - Always selecting the minimum-weight edge that connects the growing tree to an unvisited vertex - Visualizing why this greedy approach guarantees the minimum total edge weight ### Interaction Design **User Interactions:** 1. **Graph Creation Mode:** - Click on empty canvas to create vertices - Click and drag between vertices to create weighted edges - Click "Generate Random Graph" for a pre-made example 2. **Algorithm Execution:** - Click "Start Prim's Algorithm" to begin - Choose starting vertex by clicking on it (highlighted in green) - Step through the algorithm with "Next Step" button or use "Auto Run" for animation - See the current frontier edges highlighted in yellow - Watch as the minimum edge is selected (flashes, then turns blue) - Observe the MST grow with blue edges and included vertices 3. **Visual Feedback:** - Vertices change color: gray (unvisited) → green (in MST) - Edges animate: gray (not considered) → yellow (frontier) → blue (selected) - Current minimum edge pulses before selection - Running cost counter shows total MST weight - Side panel shows algorithm state and next possible edges ### Layout Description **Spatial Organization:** - **Header (24px from top):** Title and brief description - **Main Canvas (center, 800x500px):** Interactive graph visualization with 24px margins - **Control Panel (right side, 300px wide):** - Mode toggle (Create/Run) - Action buttons (Generate, Start, Step, Auto Run, Reset) - Speed slider for auto-run - Current state display - Edge frontier list with weights - Total MST cost - **Info Panel (bottom, full width):** Step-by-step explanation of current operation -
